# Post Templates — Copy, Fill, Post

Proven copy frameworks adapted for QA Mastery. Fill the [brackets]. Hook is everything — it's the only line people see before clicking "more."

---

## Framework cheat sheet
- **PAS** — Problem → Agitate → Solve. Best for pain-driven posts.
- **AIDA** — Attention → Interest → Desire → Action. Best for launches/CTAs.
- **BAB** — Before → After → Bridge. Best for transformation stories.

---

## 1. Teach post (Monday) — authority

```

1. [mistake → fix]
2. [mistake → fix]
3. [mistake → fix]

Save this for your next report.

```

## Hook formulas (first line)
- "[N] [things] most [audience] get wrong."
- "You can [common effort] and still [fail]. Here's why."
- "Last week [N] people asked for X. Today it's live."
- "Nobody tells junior testers this:"
- "I reviewed [N] bug reports. The pattern was clear:"

## CTA rules
- LinkedIn: link in **first comment**, not the post (algorithm penalizes outbound links).
- Always make the CTA a *next step*, not a *sale*: "try the free lab," "tell me what's missing."
- One CTA per post. Never two.
